Document Object Model (DOM)

Le Document Object Model (DOM) est une interface standardisée par le W3C qui permet d'accéder et de manipuler les documents HTML et XML. Il décrit la structure du document sous forme d'un arbre, où chaque élément représente un nœud de cet arbre. Ce modèle facilite la manipulation des contenus, de la structure et du style de ces documents via des scripts ou des programmes informatiques.

Évolution du DOM au sein des navigateurs web

Avant sa standardisation par le W3C, chaque navigateur web disposait de son propre modèle de document (DOM). Les développeurs devaient souvent écrire plusieurs versions de scripts pour supporter différents navigateurs.

La standardisation a permis de rendre les scripts plus universels, et elle a évolué à travers plusieurs étapes pour étendre ses fonctionnalités.

DOM 1

La première version standardisée, DOM Level 1, a été publiée en 1998. Elle a introduit un modèle arborescent pour représenter un document, avec des nœuds pour chaque élément (balises HTML ou XML). Elle permettait également de manipuler les formulaires HTML.

DOM 2

DOM Level 2 a été publié en 2000 et introduit de nouvelles fonctionnalités, telles que la possibilité d'identifier rapidement des éléments avec getElementById(), ainsi que le support des événements, des styles et des vues.

DOM 3 et 4

DOM Level 3, publié en 2004, a ajouté le support de XPath et des événements clavier, ainsi que la sérialisation des documents XML. DOM Level 4 est encore en développement.

Aspects techniques

DOM permet de manipuler la structure d'un document en parcourant son arbre d'éléments. Cependant, il peut consommer beaucoup de mémoire lorsque le document est chargé entièrement en mémoire. Pour des cas simples de lecture de documents XML, la méthode SAX est préférée car elle traite les éléments de manière séquentielle.

Événements

Les événements dans DOM permettent d'interagir avec les éléments d'une page web en fonction des actions de l'utilisateur (clics, saisie clavier, etc.).
